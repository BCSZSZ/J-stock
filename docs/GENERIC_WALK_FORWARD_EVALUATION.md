# Generic Walk-Forward Evaluation Design

## Goal

Use one unified method for all strategy combinations in evaluation results, not only MVX.

The method must:

- use rolling train->next-year-test validation,
- be fully reproducible from raw evaluation CSV,
- provide a quantitative stability score that can be audited.

## Data Input

Input is the raw evaluation CSV generated by evaluation/pos-evaluation.

Minimum required columns:

- period
- return_pct
- alpha
- sharpe_ratio
- max_drawdown_pct
- strategy identity columns (for example: entry_strategy, exit_strategy, entry_filter)

## Rolling Folds (5-year example)

Given years 2021..2025 and min_train_years=2:

- Fold A: train 2021-2022 -> test 2023
- Fold B: train 2021-2023 -> test 2024
- Fold C: train 2021-2024 -> test 2025

## Train-Time Selection Formula

For each fold, aggregate each strategy on train years and compute:

- ret_rank: percentile rank of train return_mean (higher better)
- sharpe_rank: percentile rank of train sharpe_mean (higher better)
- mdd_quality_rank: percentile rank of train mdd_mean (lower drawdown is better)

Define train score:

train_score = 0.6 * ret_rank + 0.4 * (0.5 * sharpe_rank + 0.5 * mdd_quality_rank)

Fold winner is the strategy with max train_score.

## Next-Year OOS Evaluation

On test year, compute the same type of percentile ranks for all strategies and define:

oos_utility = 0.6 * ret_rank_test + 0.4 * (0.5 * sharpe_rank_test + 0.5 * mdd_quality_rank_test)

Record for the fold winner:

- test-year rank among all strategies,
- test-year utility,
- test-year return/alpha/sharpe/mdd.

## Stability Formula (Quantitative and Auditable)

For each strategy, collect OOS utility across folds.

Let:

- mu = mean(oos_utility)
- sigma = std(oos_utility)
- p = positive_alpha_rate on OOS folds

Define stability score:

stability_score = 100 * ((1 - w) * (mu - lambda * sigma) + w * p)

Default parameters:

- lambda = 0.50 (uncertainty penalty)
- w = 0.20 (benchmark-beating consistency weight)

Interpretation:

- high mu rewards strong average OOS quality,
- high sigma is penalized (unstable behavior),
- high p rewards repeated positive alpha.

This is a lower-confidence-bound style score with an explicit consistency term.

## Year Boundary Rule (Important)

Current annual evaluation treats each year as an independent backtest period.

- 2022 starts from configured initial capital,
- not from 2021 ending capital.

So this method evaluates parameter robustness across market years, not compounding path effects.

## Code Implementation

Script:

- tools/walk_forward_stability.py

Example (all strategies identity = entry/exit/filter):

python tools/walk_forward_stability.py \
  --raw-csv strategy_evaluation/position_eval_combined_raw_YYYYMMDD_HHMMSS.csv \
  --group-cols entry_strategy exit_strategy entry_filter \
  --min-train-years 2 \
  --output-dir strategy_evaluation

Example (position-profile experiment):

python tools/walk_forward_stability.py \
  --raw-csv strategy_evaluation/position_eval_combined_raw_YYYYMMDD_HHMMSS.csv \
  --group-cols position_profile \
  --min-train-years 2 \
  --output-dir strategy_evaluation

Outputs:

- walk_forward_fold_winners_*.csv
- walk_forward_oos_panel_*.csv
- walk_forward_stability_*.csv
- walk_forward_report_*.md
