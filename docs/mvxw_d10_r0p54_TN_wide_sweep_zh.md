   - N≥9 之后 MDD 阶跃上升到 16+ %，不推荐。
4. **不要单调地放大 N 或 T**。两者收益曲线均呈"非光滑波峰"形态，提示

4. **胜率单调上升不等于收益单调上升**。N 的优化不应以胜率为目标。

### 4.2 解释：为什么"T=1.30 / N=5"是 saddle point 而非孤立峰

- **T 维**：T=1.30 与 T=0.95 是双峰，相对差距 < 1.3 %，统计上等价。
  生产应取 T=1.30 因为它有更低 MDD 和更高 Sharpe；T=0.95 留作 OOS 候选。
- **N 维**：N=5 比 N=4（−607 pp）和 N=6（−405 pp）显著更高，
  比 N=8（−192 pp）也明显更高，是真正的局部最优。
- **联合**：T=1.30 时跨 N 的最优是 N=5；N=5 时跨 T 的最优是 T=1.30/0.95。
  因此 (N=5, T=1.30) 是**真正的局部极大值点**，可以直接生产化。

### 4.3 R=0.54 体系下的"退出系统体检"

跨 T 和跨 N 两次扫描共同揭示：

1. **R1_ATRTrailing 在 R=0.54 全境失效**：胜率 0.5–17.7 %，贡献始终为 −58 ~ −156 M JPY。
   - 即使 T 调到 0.50（最积极的 trailing），R1 胜率也只有 17.7 %；
   - T 调到 2.00（最宽松的 trailing），R1 胜率掉到 0.5 %。
   - **R1 在小 R 体系下确实是"纯硬止损"，需要重新设计**。
2. **P_BiasOverheat 在 R=0.54 全境失效**：跨 T 触发 1–2 次、跨 N 触发 1–2 次，事件占比 < 0.1 %。
   - 与上一份报告结论完全一致：**应该关闭或重定义**。
3. **L2 和 N 强耦合**：N 是 L2 的"窗口长度"参数，N=5 时 L2 触发 214 次（占 8.6 %），N=10 时只剩 58 次（2.4 %）。
   - L2 损耗随 N 收敛，但 R1 损耗随 N 发散，**两者代偿**。
   - L2 不需要单独优化，跟着 N 一起调即可。
4. **T 与 P_TP2 不强耦合**：T 影响的是 trailing，但 P_TP2 触发数跨 T 跳动很小（682–774）。
   - 这说明 P_TP2（在 T×ATR 处的 take-profit）的"是否触发"主要取决于行情走势，
     而不取决于 trailing 阈值。**T 主要影响"未触发 P_TP2 的部分如何离场"**。
